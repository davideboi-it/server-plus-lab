# 2.1 Install Server Operating Systems

## 1. Hardware Compatibility and Requirements
Before installing any server OS, you must verify the physical hardware to ensure stability, performance, and vendor support.

* **Minimum OS Requirements:** The baseline hardware (CPU, RAM, Storage) needed to run the OS. Note: Enterprise workloads always require resources far beyond these minimums.
* **Hardware Compatibility List (HCL):** An official list from the OS vendor identifying certified hardware. Sticking to the HCL is mandatory for guaranteed vendor support.
* **UEFI & Secure Boot:** Modern servers utilize **UEFI** instead of legacy BIOS. **Secure Boot** is a critical security feature that prevents unsigned or malicious bootloaders and drivers from loading during the startup process.

---

## 2. Partitioning and Volume Management
* **MBR (Master Boot Record):** Legacy partitioning. Limited to 2 TB disks and 4 primary partitions.
* **GPT (GUID Partition Table):** The modern standard required for UEFI. Supports disks larger than 2 TB and up to 128 partitions.
* **LVM (Logical Volume Management):** Used in Linux to pool physical disks and resize volumes dynamically without unmounting.
* **Swap Space (Linux):** A dedicated disk area used as virtual memory. If physical RAM is exhausted, the OS moves inactive data to the Swap partition to prevent system crashes.

---

## 3. Enterprise File Systems
| File System | Associated OS | Key Features |
| :--- | :--- | :--- |
| **EXT4** | Linux (General) | Supports volumes up to 1EB; uses journaling for recovery. |
| **XFS** | Linux (Enterprise) | Default for RHEL/CentOS; optimized for large files and high-performance scaling. |
| **NTFS** | Windows Server | Default; supports ACLs (permissions) and EFS (encryption). |
| **ReFS** | Windows (Storage) | Resilient File System; features auto-healing to correct data corruption. |
| **VMFS** | VMware ESXi | Cluster-aware; allows multiple hosts to read/write to shared storage simultaneously. |

---

## 4. Server Installation Types
* **GUI (Desktop Experience):** Includes the full graphical user interface. Higher resource consumption and larger attack surface.
* **Core / Minimal Installation:** CLI-only environment. Recommended for production servers to improve security (smaller attack surface) and reduce overhead (CPU/RAM).

---

## 5. Automated and Remote Deployments
Data centers utilize automation to deploy servers at scale without physical media.

* **PXE Boot (Preboot Execution Environment):** Allows a bare-metal server to boot from the network.
    1. **DHCP Request:** Server asks for an IP and the location of boot files.
    2. **TFTP Download:** Server pulls a small bootloader image from a TFTP server.
    3. **Deployment:** The bootloader connects to a deployment server (like WDS) to install the full OS.
* **Virtual Media:** Using Out-of-Band (OOB) management tools like **iLO** or **iDRAC** to mount an ISO file from a remote workstation as a virtual drive.
* **Unattended Installation:** Uses "Answer Files" (`unattend.xml` for Windows or `Kickstart` for Linux) to provide all configuration settings automatically during setup.

---

## 6. Sysprep and Imaging (Windows Environments)
To deploy Windows quickly, admins use **Imaging** (cloning a master "Golden Image").
* **The SID Problem:** Every Windows install has a unique **Security Identifier (SID)**. Cloning without resetting the SID causes major network and Active Directory conflicts.
* **Sysprep:** The utility used to "generalize" the master image by stripping unique SIDs and names.
* **Command:** `sysprep /generalize /oobe /shutdown`.

---

## 7. Post-Installation Tasks
Once the OS is installed, technicians must complete these standard steps:
1. **Assign Hostname:** Ensure the server has a unique, descriptive name.
2. **Static IP Configuration:** Servers must have persistent IP addresses for reliable client access.
3. **Drivers & Firmware:** Install the latest vendor-certified drivers from the HCL.
4. **Domain Join:** Join the server to the central directory (Active Directory) for management.
5. **Licensing:** Activate the OS to ensure compliance and access to updates.

# 2.4 Server High Availability (HA)

## 1. High Availability Concepts
High Availability (HA) ensures that critical server-based services remain continuously accessible with minimal downtime, even in the event of hardware or software failures.
* **Fault Tolerance:** A system's ability to continue operating properly in the event of a component failure.
* **Single Point of Failure (SPOF):** Any part of a system that, if it fails, will stop the entire system from working. HA strategies aim to eliminate all SPOFs.

---

## 2. Hardware and Component Redundancy
Ensures resilience against localized, individual hardware failures. Less expensive than full server redundancy, but covers fewer failure scenarios.
* **Power Redundancy:** Servers should have **Dual Power Supplies**. For true redundancy, each supply must be plugged into a different Power Distribution Unit (PDU) and Uninterruptible Power Supply (UPS).
* **Cooling Redundancy:** Redundant, hot-swappable fans.
* **Memory and Storage:** ECC (Error-Correcting Code) memory and RAID configurations.
* **Multipath I/O (MPIO):** *Critical Enterprise Feature.* Provides multiple physical paths (cables, switches, controllers) between a server and its SAN storage. If one Fibre Channel or iSCSI switch fails, MPIO automatically routes traffic through the surviving path without dropping the connection.

---

## 3. Server Clustering
A group of two or more hosts (physical or virtual) acting as a unified system to run the same service.
* **Virtual IP / MAC:** The cluster presents a single Virtual IP and MAC address to clients. DNS points clients to this virtual IP, so they are unaware of the underlying individual nodes.
* **Active-Passive (Active/Standby):** One host actively runs the service while the other sits idle. If the active node crashes, the service fails over to the standby node.
* **Active-Active:** All hosts actively run the service simultaneously, sharing the workload. Improves both performance and redundancy.
* **Heartbeat:** A regular ping exchanged between cluster nodes to monitor their status. **Exam Tip:** The heartbeat must always run on a dedicated, isolated network interface to prevent high data traffic from causing false failovers.
* **Failover / Failback:** *Failover* is the automatic transfer of services to a healthy node. *Failback* is returning the services to the original node once it is repaired.
* **Quorum & Split-Brain:** Quorum requires a majority of nodes to agree on the cluster state. This prevents a "Split-Brain" scenario (where a broken heartbeat link causes both nodes to think the other is dead, leading both to try and take control of the data, which corrupts it). A "Witness" (a lightweight VM or file share) is often used as a tie-breaking vote.

---

## 4. Cluster Maintenance and Patching
Patching clusters requires strict procedures to maintain zero downtime.
* **Drain Roles:** Gracefully move workloads (VMs, applications) off a node before applying updates.
* **One Node at a Time:** Always update sequentially, never simultaneously.
* **Cluster-Aware Updating (CAU):** A Windows feature that automates the draining, patching, rebooting, and reintegrating of nodes one by one.
* **Version Consistency:** All nodes in a cluster must run the exact same OS build and patch level to prevent compatibility failures.

---

## 5. Load Balancing
Unlike clustering, load-balanced nodes do not share common data storage. They take turns servicing incoming client requests (typically web traffic).
* **Hardware vs. Software:** Can be a dedicated physical appliance (e.g., F5 Networks) placed in the DMZ, or software-based (e.g., HAProxy, NGINX).
* **Round Robin:** Distributes requests evenly across all available servers in sequence.
* **Most Recently Used (MRU) / Session Stickiness:** Sends traffic to the server that handled the most recent request from that client, ensuring their session (like a shopping cart) isn't lost.

---

## 6. Server Data Replication
Nodes maintain separate copies of the data, and changes are replicated across the network.
* Nodes are often geographically dispersed (Data Center A and Data Center B).
* Ideal for Active Directory Domain Controllers and SQL databases.

---

## 7. Server Network Connection Redundancy
* **NIC Teaming (LBFO):** Allows multiple Network Interface Cards on a server to work together as a single logical NIC. Provides bandwidth aggregation and failover.
    * *Switch Independent:* NICs connect to different physical switches (maximum hardware redundancy).
    * *Switch Dependent:* NICs must connect to the same switch.
* **Link Aggregation Protocol (LACP - IEEE 802.3ad):** The industry-standard protocol used to bundle multiple physical links into one logical link. Both the server NICs and the physical switch ports must be configured for LACP to work.

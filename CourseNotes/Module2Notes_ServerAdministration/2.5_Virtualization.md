# 2.5 Server Virtualization and Cloud Computing

## 1. Virtualization Core Concepts
Virtualization abstracts physical hardware, allowing multiple Virtual Machines (VMs) to run simultaneously on a single physical server. 
* **Host:** The underlying physical server that provides the hardware resources (CPU, RAM, Disk, Network).
* **Guest:** The Virtual Machine (VM) running its own isolated Operating System and applications.
* **Hypervisor (VMM - Virtual Machine Monitor):** The software layer that manages, provisions, and allocates physical resources to the VMs.

### Hypervisor Types
* **Type 1 (Bare-Metal):** Installs directly onto the physical server hardware. Highly efficient, secure, and used in enterprise Data Centers.
    * *Examples:* VMware ESXi, Microsoft Hyper-V (Server role), Proxmox (KVM), Citrix XenServer.
* **Type 2 (Hosted):** Installs as an application on top of an existing host OS (like Windows 11 or macOS). Primarily used for testing, labs, and development.
    * *Examples:* VMware Workstation, Oracle VirtualBox.

## 2. Virtual Networking
VMs communicate through virtualized network components that mimic physical networking gear.
* **vNIC (Virtual NIC):** The software-based network adapter assigned to the VM.
* **vSwitch (Virtual Switch):** Software switch inside the hypervisor that routes traffic between VMs and connects them to the physical network via the physical NIC (pNIC).

### VM Connection Modes
* **Bridged (Direct Access):** The vNIC connects directly to the physical LAN. The VM gets its own IP address from the physical DHCP server and acts exactly like a physical machine on the network.
* **NAT (Network Address Translation):** The VM gets a private, internal IP from the hypervisor. It shares the Host's IP address to access the external network. External devices cannot initiate connections to the VM without port forwarding.
* **Host-Only:** The VM can only communicate with other VMs on the same host and the Host OS itself. It has no access to the external physical network or the Internet (used for isolated testing).

## 3. Resource Allocation and Provisioning
* **vCPU:** Virtual processors assigned to a VM. Over-provisioning vCPUs (assigning more total vCPUs to VMs than physical cores available) is common, but extreme over-provisioning causes "CPU Contention" (high CPU Ready times).
* **Memory (RAM):** Memory allocated to the VM. Features like *Memory Ballooning* allow the hypervisor to dynamically reclaim unused RAM from idle VMs to give to busy VMs.
* **Virtual Disks:** VMs store data in single, large files on the host system.
    * *Formats:* `.vmdk` (VMware), `.vhdx` (Hyper-V), `.ova` (Open Virtual Appliance - used for exporting/importing pre-configured VMs).
    * **Thick Provisioning:** The entire disk space (e.g., 60GB) is pre-allocated on the physical storage immediately upon creation. Faster performance, but uses up space instantly.
    * **Thin Provisioning:** The virtual disk only takes up as much physical storage as the actual data inside it, growing on-demand up to its maximum limit. Saves space but requires strict monitoring to prevent the physical datastore from filling up.

## 4. Virtual Machine Features and Management
* **Snapshots (Checkpoints):** Captures the exact state, data, and configuration of a VM at a specific point in time. Crucial before applying patches or updates (allows instant rollback). *Note: Snapshots are NOT long-term backups.*
* **Guest Integration Tools:** Software installed inside the Guest OS (e.g., VMware Tools, Hyper-V Integration Services) that provides optimized drivers, better mouse/display performance, and time synchronization with the host.
* **vTPM (Virtual Trusted Platform Module):** A virtualized security chip that allows the Guest OS to use advanced security features like Secure Boot and BitLocker disk encryption.
* **Centralized Management:** Enterprise interfaces used to manage clusters of hypervisors and thousands of VMs (e.g., VMware vCenter Server Appliance - VCSA, Microsoft System Center Virtual Machine Manager - SCVMM).

## 5. Cloud Computing Concepts
Cloud computing extends virtualization by automating resource delivery over the Internet on a pay-as-you-go model, shifting costs from CapEx (buying physical servers) to OpEx (monthly operational costs).

### Cloud Deployment Models
* **Public Cloud:** Infrastructure owned and managed by third-party providers (AWS, Azure, Google Cloud). Resources are shared among multiple tenants over the public internet.
* **Private Cloud:** Infrastructure dedicated exclusively to a single organization, providing maximum control and security (can be on-premises or hosted by a third party).
* **Hybrid Cloud:** A combination of Public and Private clouds bound together by technology that allows data and applications to be shared between them (e.g., keeping sensitive databases on-prem, but moving web servers to AWS).
* **Multi-Cloud:** Using multiple different public cloud providers (e.g., using AWS for storage and Azure for Active Directory) to avoid vendor lock-in.

### Cloud Service Models (The "As-A-Service" Stack)
* **IaaS (Infrastructure as a Service):** The provider gives you the raw virtualized hardware (VMs, storage, networks). You are responsible for managing the OS, middleware, and applications (e.g., AWS EC2).
* **PaaS (Platform as a Service):** The provider manages the hardware and the OS. You focus solely on deploying your application code or databases (e.g., Heroku, Azure App Service).
* **SaaS (Software as a Service):** The provider manages everything. You just consume the software via a web browser (e.g., Microsoft 365, Salesforce).

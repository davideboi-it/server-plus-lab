# 2.7 Asset Management and Documentation

## 1. Introduction to Server Asset Management
Server asset management involves tracking, maintaining, and optimizing physical and virtual server resources. 
* It ensures that hardware and software assets are accounted for, compliant, and efficiently utilized. 
* Proper management supports IT governance, budgeting, lifecycle management, and incident response.

---

## 2. Key Components of Asset Management
Effective management requires strict oversight of what the organization owns and how it is used.
* **Inventory Tracking:** Catalog physical servers, virtual machines, network interfaces, and peripheral hardware. *Exam Tip: Physical servers are often tracked using RFID tags or Barcode labels (Asset Tags) physically attached to the chassis.*
* **Configuration Documentation:** Record server roles, OS versions, installed software, patch levels, IP addresses, and VLANs.
* **Lifecycle Management:** Track asset acquisition dates, warranty expirations, end-of-life status, and refresh cycles. *Exam Tip: The final stage of the lifecycle is "Decommissioning," which strictly requires secure data destruction (e.g., shredding hard drives) before physical disposal.*
* **License Compliance:** Monitor installed software against purchased licenses to avoid under- or over-licensing (which can result in heavy vendor audits and fines).

---

## 3. Server Documentation Essentials
Good documentation is the difference between a quick fix and a prolonged outage during a crisis.
* **System Diagrams:** Visual layouts of rack positions, cabling, network topology, and VLAN assignments.
* **Runbooks / SOPs (Standard Operating Procedures):** Step-by-step instructions for startup, shutdown, failover, and patch management.
* **Access Control Lists (ACLs):** Track who has admin access, credentials management, and role-based permissions.
* **Maintenance Records:** Log updates, patches, incidents, part replacements, and configuration changes.

---

## 4. Tools for Asset Management and Documentation
Relying on memory or messy spreadsheets is not acceptable in enterprise environments.
* **Asset Management Systems:** Tools like GLPI, Snipe-IT (open-source), Lansweeper, ServiceNow, and SolarWinds.
* **CMDBs (Configuration Management Databases):** Advanced databases that integrate with ITSM (IT Service Management) tools for change tracking and impact analysis. They show the *relationships* between assets (e.g., "If Switch A goes down, Server B and C will lose connection").
* **Documentation Platforms:** Centralized wikis like Confluence, OneNote, IT Glue, or Markdown repositories.
* **Automation Tools:** Using Ansible, Puppet, or scripts to automatically document system states without human intervention.

---

## 5. Best Practices
* **Centralization:** Keep asset and documentation records centralized, version-controlled, and regularly updated.
* **Standardization:** Use consistent naming conventions (e.g., `Location-Role-Number` like `GVA-WEB-01`) and standard fields to ensure consistency.
* **Auditing:** Schedule periodic audits to validate hardware presence and configuration accuracy against the documentation.
* **Security:** Make documentation accessible but secure using RBAC (Role-Based Access Control) and ensure the documentation itself is backed up.

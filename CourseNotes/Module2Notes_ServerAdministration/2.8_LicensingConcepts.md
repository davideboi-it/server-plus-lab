# 2.8 Licensing Concepts

## 1. Introduction to Server and Software Licensing
Licensing governs how software and server resources can be legally used, deployed, and accessed within an organization.
* It protects vendor intellectual property while giving customers structured access to features and support.
* Choosing the right model ensures legal compliance, cost efficiency, and seamless scalability.
* **EULA (End-User License Agreement):** The legal contract between the software author and the user establishing the purchaser's right to use the software. Always review the EULA for specific limitations.

---

## 2. Common Licensing Models
Different software requires different purchasing strategies based on hardware size or user count.
* **Per-Core / Per-Socket:** Licenses tied directly to the physical hardware. Modern enterprise software (like Windows Server or SQL Server) is heavily shifting to **Per-Core** licensing to account for high-density processors.
* **Per-Instance / Per-VM:** License applies to each running Virtual Machine, regardless of the underlying physical hardware.
* **Per-Concurrent User:** Limits the number of simultaneous active sessions (e.g., You have 100 employees, but only 20 can be logged into the database at the exact same time).
* **Site-Based:** A flat-rate license allowing unlimited use within a single physical location or organization.
* **Volume Licensing:** Bulk purchasing model providing a single master key to activate multiple installations centrally across an enterprise.

---

## 3. License Types and Portability (*Exam Focus*)
How a license is purchased determines if it can be moved to a new server when hardware fails.
* **OEM (Original Equipment Manufacturer):** The software comes pre-installed on the server hardware. *Crucial:* OEM licenses are permanently tied to that specific motherboard. If the server hardware dies, the license dies with it. It cannot be transferred.
* **Retail / Commercial:** Standard licenses that can be uninstalled from an old server and legally moved to a newly purchased server.
* **Open-Source (FOSS):** Software permitted for use, modification, and redistribution (e.g., Linux, Apache).
* **BYOL (Bring Your Own License):** A cloud computing model that lets you apply your existing on-premises volume licenses to virtual machines hosted in public clouds (like AWS or Azure) to save costs.

---

## 4. License Validation and Enforcement
Vendors use various mechanisms to ensure organizations aren't using more software than they paid for.
* **Activation / Call-Home:** The software securely connects to the vendor's servers over the internet to verify the key and compliance.
* **License Server (e.g., Microsoft KMS):** Instead of each individual server contacting the vendor over the internet, servers contact a local centralized Key Management Service (KMS) server inside your datacenter to stay activated.
* **True-Up:** A yearly audit process in Enterprise Agreements (EA) where a company counts its actual software usage and pays the difference for any extra installations added during the year.
* **Node-Locked:** Software that is cryptographically bound to a specific physical hardware component (like a MAC address or CPU serial number).

---

## 5. License Version Compatibility
* **Backward Compatibility (Downgrade Rights):** Many enterprise licenses allow you to buy the newest version (e.g., Windows Server 2022) but legally install an older version (e.g., Windows Server 2019) if your legacy applications require it.
* **Forward Compatibility:** Licenses generally do *not* permit use of future versions unless you have an active subscription or maintenance plan.

---

## 6. License vs. Maintenance and Support
It is vital to understand the difference between owning the software and getting help for it.
* **License:** Grants the legal right to use the software (often a one-time perpetual purchase).
* **Maintenance and Support / Software Assurance (SA):** A recurring fee (usually annual) that grants access to security patches, major version upgrades, and technical phone support. *If maintenance expires, you can still use the software indefinitely, but you lose access to updates and help.*
* **Subscription Model:** (e.g., Microsoft 365). Bundles both the license and support into one recurring fee. If you stop paying, the software stops working completely.

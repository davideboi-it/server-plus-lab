# 2.3 Server Functions and Features

## 1. Common Server Services & Protocols
Servers centralize resources to improve manageability, scalability, and security.

### Web Services
* **HTTP (Port 80):** Standard web traffic, unencrypted.
* **HTTPS (Port 443):** Secure web traffic using SSL/TLS encryption.

### File & Print Services
* **FTP (Ports 20, 21):** File Transfer Protocol. Port 21 for control, Port 20 for data.
* **SMB/CIFS (Port 445):** Server Message Block. Used for Windows file and printer sharing.
* **NFS (Port 2049):** Network File System. Common for Linux/Unix file sharing.
* **TFTP (Port 69):** Trivial FTP. Used for simple transfers (like PXE booting) without authentication.
* **SFTP (Port 22):** Secure FTP using SSH for encryption.

### Email Services
* **SMTP (Port 25):** Simple Mail Transfer Protocol. Used for sending email.
* **POP3 (Port 110):** Post Office Protocol v3. Used for retrieving email (downloads to client).
* **IMAP4 (Port 143):** Internet Message Access Protocol. Used for retrieving email (syncs with server).

---

## 2. Directory Services and Authentication
Directories handle Identity Management and Authentication.

* **DNS (Port 53):** Domain Name System. Resolves hostnames to IP addresses (UDP for queries, TCP for zone transfers).
* **DHCP (Ports 67, 68):** Dynamic IP assignment. Port 67 for the server, Port 68 for the client.
* **LDAP (Port 389):** Lightweight Directory Access Protocol. Used for directory queries.
* **LDAPS (Port 636):** Secure LDAP over SSL/TLS.
* **Kerberos (Port 88):** Authentication protocol for Windows Active Directory.
* **RADIUS (Ports 1812, 1813):** Remote Authentication Dial-In User Service. Used for network access security.

---

## 3. Server Storage Management
* **MBR (Master Boot Record):** Legacy partitioning; max 4 primary partitions; max 2 TB size.
* **GPT (GUID Partition Table):** Modern standard; required for UEFI; supports up to 128 partitions and massive disk sizes (ZB/EB).
* **Disk Quotas:** Restricts user storage space to prevent resource exhaustion.
* **Virtual Memory (Swap/Page File):** Used as overflow when physical RAM is full. Windows uses `pagefile.sys`; Linux uses a swap partition.
* **Data Deduplication:** Eliminates redundant data blocks to save space.

---

## 4. Server Monitoring and Baselining
**Baselining** defines the "normal" performance of a server to help identify anomalies.

* **CPU Metrics:** Monitor utilization (threshold >80%) and queue length.
* **Memory Metrics:** Monitor utilization and page faults (excessive paging indicates low RAM).
* **Disk Metrics:** Monitor IOPS (Input/Output Per Second), Throughput, and Latency.
* **Network Metrics:** Monitor bandwidth usage, errors, and drops.
* **SNMP (Ports 161, 162):** Simple Network Management Protocol. Port 161 for agents, Port 162 for traps (alerts).

---

## 5. Event Logs and Centralized Logging
* **Syslog (Port 514):** Standard protocol for message logging. Often used to send server logs to a central repository.
* **SIEM (Security Information and Event Management):** Software that aggregates and analyzes logs from different sources to detect security threats.
* **Log Rotation:** The process of archiving old logs and starting new ones to save disk space (e.g., `logrotate`).

---

## 6. Remote Administrative Interfaces
* **SSH (Port 22):** Secure Shell. Encrypted CLI access for Linux/Unix and network devices.
* **Telnet (Port 23):** Legacy CLI access. Unencrypted and insecure.
* **RDP (Port 3389):** Remote Desktop Protocol. Graphical access for Windows servers.
* **VNC (Port 5900):** Virtual Network Computing. Platform-independent graphical remote access.
* **KVM over IP:** Hardware-based remote access (keyboard, video, mouse) that works even if the OS is not responding.
* **Out-of-Band (OOB) Management:** Dedicated management interfaces like HP iLO or Dell iDRAC (usually accessed via HTTPS/443).

# 1.6 Server Storage Technologies and RAID

## 1. Drive Interfaces and Form Factors
To accurately configure server storage, you must distinguish between how a drive communicates (Protocol/Interface) and how it physically fits into the chassis (Form Factor).

### Interfaces / Protocols
* **SATA:** Entry-level interface, max speed around 6 Gbps (~600 MB/s). Typically used for bulk storage where speed is not the primary concern.
* **SAS (Serial Attached SCSI):** High-performance enterprise interface. Crucially, SAS supports dual-porting, allowing the drive to connect to two separate storage controllers simultaneously for hardware redundancy (Multipath I/O).
* **NVMe (Non-Volatile Memory Express):** Operates over the PCIe interface, bypassing legacy storage controllers. It offers ultra-low latency and massive parallelism by communicating almost directly with the CPU.

### Form Factors
* **LFF (3.5") & SFF (2.5"):** Large Form Factor and Small Form Factor, used for standard physical HDDs and SSDs.
* **M.2:** Gumstick-shaped NVMe drives installed directly onto the motherboard. They are highly susceptible to physical damage if mishandled and are generally **not** hot-swappable.
* **U.2:** Looks like a standard 2.5" drive but utilizes a specialized connector to provide NVMe-over-PCIe speeds. Crucially, U.2 drives **are hot-swappable** in front-facing server drive bays.

## 2. RAID Concepts and Levels
RAID provides performance, redundancy, or both. **RAID is NOT a backup!**

### Hardware vs. Software RAID
* **Hardware RAID:** A dedicated PCIe controller card manages the RAID calculations, offloading the work from the CPU. These cards feature onboard cache and a Battery Backup Unit (BBU) to protect pending writes during a power outage.
* **Hot Spare:** A dedicated, unused drive plugged into the array. If an active drive fails, the RAID controller automatically grabs the hot spare and begins rebuilding the array immediately without human intervention.

### Standard RAID Levels
* **RAID 0 (Striping):** Data is split across drives for maximum speed. Zero fault tolerance; if one drive dies, the entire array is destroyed. *Minimum 2 disks.*
* **RAID 1 (Mirroring):** Data is duplicated exactly to another drive. Survives 1 disk failure. Usable capacity is exactly 50%. *Minimum 2 disks.*
* **RAID 5 (Striping with Parity):** Data and parity blocks are distributed across all disks. Survives exactly 1 disk failure. You lose the capacity of 1 drive for the parity overhead. *Minimum 3 disks.*
* **RAID 6 (Striping with Dual Parity):** Like RAID 5, but writes two sets of parity. Survives 2 simultaneous disk failures. Write performance is slower due to double parity calculations. *Minimum 4 disks.*
* **RAID 10 (Stripe of Mirrors):** Mirrors data first, then stripes it. Provides the highest performance and fault tolerance. Survives multiple failures (as long as they aren't in the same mirrored pair). *Minimum 4 disks.*

## 3. Storage Architectures: DAS, NAS, and SAN
Choosing the correct storage architecture based on budget and performance requirements is heavily tested.

* **DAS (Direct-Attached Storage):** Storage directly connected to a single server (internal drives or a direct-cabled external enclosure). It provides **Block-level** access. It is an isolated island of storage and cannot be shared across a network.
* **NAS (Network-Attached Storage):** A dedicated storage appliance connected to the standard LAN. It provides **File-level** access. Clients connect to share files using Layer 7 protocols like SMB/CIFS (Windows) or NFS (Linux). Easy to deploy, but not suitable for high-performance databases or virtual machine clusters.
* **SAN (Storage Area Network):** A dedicated, high-speed network built exclusively for storage traffic. It provides **Block-level** access. The server's OS is tricked into treating SAN storage as if it were a local physical hard drive. The total storage array is divided into logical chunks called **LUNs (Logical Unit Numbers)**, which are assigned to specific servers.

## 4. SAN Protocols
Servers connect to a SAN using an **HBA (Host Bus Adapter)**, which is the storage network equivalent of a standard NIC.

* **Fibre Channel (FC):** High-performance enterprise protocol running over dedicated optical fiber switches. Instead of IP/MAC addresses, FC routing uses **WWNs (World Wide Names)**.
* **FCoE (Fibre Channel over Ethernet):** Encapsulates Fibre Channel frames inside standard Ethernet packets, allowing storage and network traffic to share the same 10G+ copper or fiber cables.
* **iSCSI:** Block-level storage that runs over standard TCP/IP networks. Much cheaper than Fibre Channel. Devices are identified using an **IQN (iSCSI Qualified Name)**.
* **NVMe/TCP & RoCE:** Emerging standards allowing ultra-fast NVMe storage to run over Ethernet. RoCE (RDMA over Converged Ethernet) allows the network adapter to bypass the CPU entirely, drastically lowering latency.

## 5. Enterprise File Systems Cheat Sheet
Matching the correct file system to the environment is critical for deployment:
* **Windows Server:** **NTFS** (Default, supports file-level encryption/permissions) and **ReFS** (Resilient File System, features auto-healing and is used for Storage Spaces).
* **Linux:** **EXT4** (Default general purpose) and **XFS** (High-performance enterprise standard).
* **VMware ESXi:** **VMFS** (Proprietary clustered file system that allows multiple hypervisors to read/write to the same SAN LUN simultaneously).
* **FreeBSD / Custom NAS:** **ZFS** (Advanced file system featuring copy-on-write, massive storage pools, and software-based volume management).
